# Security Policy

## Reporting a Vulnerability
branPlease report security vulnerabilities responsibly.

Preferred method:
- Use GitHub’s private security reporting (Security tab → “Report a vulnerability”), if enabled.

If private reporting is not available, please open an issue with **no sensitive details** and request a private channel for disclosure.

## What to Include

- Description of the vulnerability
- Steps to reproduce (minimal PoC if safe)
- Affected paths/components and versions
- Impact assessment

## Response Targets

- Acknowledgement: within 2 business days
- Status update: within 7 business days
